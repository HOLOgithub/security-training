📘 セキュリティ教育資料

情報セキュリティの基本を学ぼう

この資料を通じて、会社の情報資産を守るための基本知識を身につけましょう

1. はじめに

本資料の目的とセキュリティの重要性について

本資料の目的

新入社員の皆様に情報セキュリティの基本知識を身につけていただき、日常業務での適切な行動を促進することを目的としています。

セキュリティ事故の影響

セキュリティ事故は、会社の信頼失墜、経済的損失、法的責任など、深刻な影響をもたらします。一度失った信頼を回復するには長い時間がかかります。

社員一人ひとりの責任

情報セキュリティは特定の部署だけの問題ではありません。社員一人ひとりの意識と行動が会社を守る重要な盾となります。

重要なポイント

セキュリティは「やらされるもの」ではなく、「会社と自分を守るために必要なもの」という意識を持つことが大切です。

2. なぜ情報セキュリティが必要なのか

現代社会におけるセキュリティ脅威の実態

社会情勢

サイバー攻撃の増加

  • • 年々巧妙化するフィッシング攻撃
  • • ランサムウェアによる企業活動停止
  • • 標的型攻撃による機密情報窃取
  • • SNSを悪用した情報収集

情報漏洩事件の実例

  • • 大手企業の個人情報流出（数百万件規模）
  • • 医療機関の患者データ漏洩
  • • 金融機関のシステム障害
  • • 官公庁の機密文書流出

実際の企業事例

事例1: メール誤送信による顧客情報漏洩

宛先を間違えて顧客リストを外部に送信。結果：謝罪、システム改修費用1000万円、顧客離れ

教訓：送信前の確認の重要性

事例2: USBメモリの紛失

営業担当者が顧客データ入りUSBを電車内で紛失。結果：全顧客への謝罪、契約解除

教訓：私物デバイスでの業務データ持ち出し禁止

中小企業も例外ではない

狙われる理由

  • • セキュリティ対策が不十分
  • • 大企業への踏み台として利用
  • • 専門知識を持つ人材が不足

被害の深刻さ

  • • 復旧費用が経営を圧迫
  • • 信頼失墜による取引停止
  • • 事業継続困難に陥るケースも

3. 日常業務に潜むセキュリティリスク

身近なところに潜む様々な危険

身近なリスク例

書類やノートの放置

危険な行為
  • • 机の上に書類を置いたまま外出
  • • 会議室にホワイトボードの内容を残す
  • • プリンターに印刷物を放置
  • • 廃棄書類をそのままゴミ箱へ
想定される被害
  • • 競合他社への情報流出
  • • 顧客の個人情報漏洩
  • • 新商品開発情報の盗取
  • • 営業戦略の漏洩

会話からの情報漏洩

注意すべき場所
  • • エレベーター内での会話
  • • 喫煙所・休憩室での雑談
  • • 電車内での電話
  • • レストランでの商談
リスクの例
  • • 同業他社の社員が聞いている
  • • SNSで拡散される可能性
  • • 意図しない第三者への情報提供
  • • 録音される危険性

USBメモリや私物PCの使用

問題となる行為
  • • 私物USBでのデータ移動
  • • 自宅PCでの業務作業
  • • クラウドサービスへの無断保存
  • • 私物スマートフォンでの撮影
危険性
  • • マルウェア感染のリスク
  • • データの紛失・盗難
  • • セキュリティ管理外での情報保存
  • • アクセス権限の管理不能

フィッシングメールや不審な添付ファイル

典型的な手口
  • • 銀行を装ったパスワード要求
  • • 宅配業者を装った不在通知
  • • 上司を装った緊急の指示
  • • ウイルス感染を警告する偽メール
被害例
  • • IDやパスワードの盗取
  • • マルウェアによるシステム感染
  • • 不正送金や詐欺被害
  • • 機密情報の外部送信

SNSやチャットでの不用意な発言

問題となる投稿
  • • 新商品開発の進捗をほのめかす
  • • 顧客名や案件名を具体的に記載
  • • 会社の内部情報を愚痴として投稿
  • • 職場の写真に機密情報が写り込み
影響
  • • 情報の意図しない拡散
  • • 会社の信頼度低下
  • • 顧客との関係悪化
  • • 法的責任の発生

4. 基本ルール（必ず守るべきこと）

全社員が遵守すべき6つの重要ルール

🔒 機密情報は絶対に社外に持ち出さない

禁止事項

  • • 業務データの私物デバイスへの保存
  • • 個人メールアドレスへの業務メール転送
  • • 許可されていないクラウドサービスの使用
  • • 業務書類の自宅への持ち帰り

適切な対応

  • • 会社指定のVPN環境でのリモートアクセス
  • • 承認されたクラウドサービスの利用
  • • セキュアな環境でのデータ共有
  • • 業務完了後の確実なデータ削除

📝 ノートや資料を机に置きっぱなしにしない

クリアデスク原則

  • • 離席時は全ての書類を収納
  • • 机上に機密情報を残さない
  • • 共有スペースでの書類放置禁止
  • • プリンターでの印刷物の即座回収

実践方法

  • • 鍵付きキャビネットでの保管
  • • 作業終了時の机上確認習慣
  • • 一時的な離席でも書類を片付け
  • • ホワイトボードの内容消去

🗣️ 社内情報を安易に口にしない

注意すべき場所

  • • エレベーター・廊下
  • • 喫煙所・休憩室
  • • 外出先（レストラン、電車など）
  • • オンライン会議の前後

対策

  • • 機密性の高い話は会議室で
  • • 第三者がいる場所での業務会話回避
  • • 固有名詞（顧客名・商品名）の使用注意
  • • 電話での機密情報のやり取り禁止

💻 パソコンのロックは離席時に必ず実施

ロックが必要な場面

  • • トイレ・休憩での短時間離席
  • • 会議参加のための移動
  • • 昼休みや外出時
  • • 他の人に話しかけられた時

ロック方法

  • • Windows: Windowsキー + L
  • • Mac: Control + Command + Q
  • • 自動ロック設定（5分以内推奨）
  • • スクリーンセーバーのパスワード設定

📦 ゴミ箱にも注意（シュレッダー使用）

シュレッダー対象

  • • 顧客情報が記載された書類
  • • 印刷ミスした機密文書
  • • 手書きのメモや付箋
  • • CDやUSBメモリなどの記録媒体

適切な廃棄方法

  • • クロスカット方式での細断
  • • 機密レベルに応じた廃棄区分
  • • 電子媒体の物理破壊
  • • 廃棄記録の適切な管理

✉ 不審なメールを開かない、報告する

不審メールの特徴

  • • 差出人が不明・偽装されている
  • • 緊急性を装い行動を促す内容
  • • 不自然な日本語や文法ミス
  • • 怪しいリンクや添付ファイル

対処法

  • • 開封せずに情報システム部門へ報告
  • • リンクのクリック・添付ファイル実行禁止
  • • 個人情報の入力絶対禁止
  • • 疑わしい場合は必ず相談

重要な心構え

これらのルールは「面倒な制約」ではなく、「会社と自分を守るための重要な習慣」です。毎日の実践により、自然に身につくよう心がけましょう。

5. ケーススタディ（〇✕クイズ形式）

実際のシーンで正しい判断ができるかチェックしよう

以下の場面で、適切な行動を〇、不適切な行動を✕で答えてください。

問題1: ランチタイム

「同僚とランチを食べながら、隣の席で進行中の新商品開発プロジェクトについて詳しく話し合った。レストランは社内の人しかいないようだった。」

正解: ✕ 不適切

レストランなどの公共の場所では、どこで誰が聞いているか分かりません。社内の人しかいないように見えても、他の客や店員が聞いている可能性があります。機密性の高い話は適切な場所で行いましょう。

問題2: 休憩時間

「5分間だけ休憩室に行くため、パソコンの画面に顧客リストを表示したまま席を離れた。すぐ戻る予定だったのでロックしなかった。」

正解: ✕ 不適切

時間の長短に関わらず、席を離れる際は必ずパソコンをロックする必要があります。5分間でも十分に機密情報を見られたり、操作されたりする可能性があります。Windowsキー+Lで簡単にロックできます。

問題3: メール対応

「銀行を名乗るメールで『不正アクセスが検出されたため、24時間以内にパスワードを変更してください』という内容が届いた。リンクをクリックせず、直接銀行のホームページにアクセスして確認した。」

正解: 〇 適切

メール内のリンクをクリックせず、独自に公式サイトにアクセスして確認する行動は非常に適切です。フィッシングメールの多くは緊急性を装ってリンクをクリックさせようとします。疑わしい場合は必ず別途確認しましょう。

問題4: 在宅勤務

「在宅勤務中、作業効率を上げるため、顧客データを私物のUSBメモリにコピーした。作業終了後、USBメモリのデータは削除した。」

正解: ✕ 不適切

私物のUSBメモリに業務データをコピーすることは、たとえ後で削除したとしても情報漏洩のリスクが高く、原則禁止です。在宅勤務では会社が提供するVPN環境や承認されたクラウドサービスを利用しましょう。

問題5: 書類廃棄

「印刷ミスした顧客名簿をそのままゴミ箱に捨てた。どうせ印刷失敗で読めない部分が多いから問題ないと思った。」

正解: ✕ 不適切

印刷ミスであっても、顧客情報が含まれる書類は必ずシュレッダーで破棄する必要があります。読めない部分があっても、読める情報から個人が特定される可能性があります。機密情報は必ず適切に廃棄しましょう。

問題6: SNS投稿

「新しいプロジェクトが始まって嬉しかったので、『新しい案件でワクワクしてます！頑張ります！』とSNSに投稿した。具体的な内容は書かなかった。」

正解: 〇 適切

具体的な案件名や顧客名、プロジェクトの詳細を書かずに、一般的な表現で仕事への意欲を表すのは問題ありません。ただし、会社のSNSガイドラインがある場合はそれに従い、投稿前に内容を再確認する習慣をつけましょう。

学習のポイント

全問正解できましたか？間違えた問題があれば、なぜその行動が不適切なのかをしっかり理解し、実際の業務で同じミスを繰り返さないよう注意しましょう。

6. 万が一トラブルが起きたら？

緊急時の正しい対処法と連絡体制

1. すぐに連絡

上司・情報システム担当へ即座に報告

2. 隠さない

事実を正確に報告し、隠蔽しない

3. 自己判断禁止

独断での対処は避け、指示を仰ぐ

情報漏洩・セキュリティインシデント

発生例

  • • メールの誤送信
  • • USBメモリ・書類の紛失
  • • 不正アクセスの発見
  • • マルウェア感染の疑い
  • • 個人情報の漏洩

即座に行うこと

  • • 上司・情シス部門への連絡
  • • 被害拡大防止措置
  • • 状況の詳細記録
  • • 関係者への情報共有
  • • 外部機関への届出準備

フィッシングメール・不審メール

対処手順

  1. メールを開かない・リンクをクリックしない
  2. 添付ファイルを実行しない
  3. 情報システム部門に報告
  4. 同僚への注意喚起を検討
  5. 似たようなメールの確認

もしクリックしてしまったら

  • • 即座にネットワークから切断
  • • パスワードの変更
  • • システム管理者への緊急連絡
  • • ウイルススキャンの実行
  • • 被害状況の確認

システム障害・機器トラブル

よくあるトラブル

  • • パソコンの動作不良
  • • ネットワーク接続エラー
  • • データファイルの破損
  • • プリンター等の機器故障
  • • ソフトウェアの異常終了

基本対応

  • • 再起動前に作業内容を保存
  • • エラーメッセージのスクリーンショット
  • • 情報システム部門への連絡
  • • 代替手段での業務継続検討
  • • 復旧までの影響範囲確認
